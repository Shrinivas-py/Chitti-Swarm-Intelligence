Medical Diagnosis Aid — Full Implementation Deep Dive

A clinician inputs patient symptoms. Chitti spawns 5 specialist-personality agents. They independently generate differential diagnoses from shared MCP memory, score each other's reasoning, and evolve — agents with poor diagnostic logic are pruned. Final output: ranked differential with confidence % and red flags. Patient data never leaves the hospital.

System Flow

Clinician Input
Symptoms, vitals, history, medications
→
MCP Memory Init
Problem stored as shared context pool
→
5 Specialists Generate
Each agent from different clinical lens
→
Score + Prune
LLM scores diagnostic quality 1–10
→
Synthesise
Ranked differential + red flags + next tests

Specialist Agent Personalities

CARDIOLOGIST-AGENT
Leads with cardiac causes — chest pain radiation, arrhythmia indicators, BP patterns, risk factors (age, diabetes, smoking, family history). Catches rare cardiac presentations.
temp: 0.7 · style: exploratory
NEUROLOGIST-AGENT
Flags neuro red flags — headache patterns, focal deficits, consciousness changes, TIA indicators. Systematic and precise. Prevents missing acute neurological events.
temp: 0.3 · style: systematic
INTERNIST-AGENT
Broad differential — GI, metabolic, infectious, autoimmune. Reads what cardio and neuro agents produced and looks for the gaps. Convergent synthesis role.
temp: 0.5 · style: convergent
SKEPTIC-AGENT
Challenges every diagnosis — what doesn't fit? What symptom is being ignored? What assumption was made? Prevents premature diagnostic closure. The most valuable agent.
temp: 0.5 · style: adversarial
RARE-DISEASE-AGENT
High-temperature creative agent — hunts for zebras that fit the symptom pattern when horses have been ruled out. The one that catches the diagnosis everyone missed. Only used if standard differentials score below 7.0.
temp: 0.9 · style: creative · activated: conditionally

Legal Document Review — How It Works

Upload a contract. Three agents read it simultaneously via shared MCP memory: a risk-spotter, a precedent-checker, and a client-advocate. Each flags different issues. The skeptic challenges false alarms. Final output: prioritised risk register with severity scores and recommended clause edits.
RISK-SPOTTER AGENT
Hunts for unlimited liability clauses, missing IP assignment, unilateral termination rights, penalty clauses. Outputs risk severity 1–10 per clause.
temp: 0.4 · systematic
PRECEDENT AGENT
Cross-references clauses against case law. Contextualises risk — a clause that reads dangerous may be standard in the jurisdiction. Calibrates risk scores from the risk-spotter.
temp: 0.3 · systematic
CLIENT ADVOCATE
Reads the contract from the client's business reality. Identifies clauses that are theoretically acceptable but operationally dangerous given the client's specific model.
temp: 0.6 · convergent
SKEPTIC AGENT
Challenges both alarmism and false reassurance. Prevents over-lawyering AND under-reaction. Produces the calibrated final risk register.
temp: 0.5 · adversarial
RISK-SPOTTER: "Clause 14.3 — unlimited liability with no cap. RISK: 9/10. This could bankrupt the client on a single contract dispute."
PRECEDENT: "Clause 14.3 mirrors Hadley v Baxendale standard. Courts interpret this narrowly — actual causation required. Revise risk to 6/10."
CLIENT ADVOCATE: "Client's net margin is 4%. Even a £200K dispute under 14.3 is existentially dangerous. Precedent argument doesn't protect against litigation cost. Risk stays at 8/10 for THIS client."
⬡ SYNTHESIS: Clause 14.3 — RISK 7/10. Recommend: add £500K liability cap. Precedent limits extreme exposure but litigation cost alone is dangerous at client's margin. Draft redline provided.

Startup Strategy — How It Works

Founder describes their startup idea. Chitti runs a VC agent, operator agent, and customer agent simultaneously. They stress-test market size, unit economics, and real user pain. Final output: go/no-go with the 3 critical assumptions to validate and suggested experiments.
VC-MINDSET AGENT
Is this a billion-dollar market? Can it scale 10x without proportional cost increase? What's the defensible moat in year 3? Thinks in fund returns, not unit economics.
OPERATOR AGENT
What breaks first at 1000 users? What's the real CAC? What's the support nightmare at scale? Thinks in hiring, processes, and the things founders ignore until it's too late.
CUSTOMER AGENT
Would a real person pay for this today without being asked nicely? What's the switching cost from the current solution? Is this a painkiller or a vitamin?
DEVIL'S ADVOCATE
Who already does this better? What's the single assumption this entire business rests on? What does the market look like if that assumption is wrong?

Code Architecture Review — How It Works

Paste your codebase or architecture description. Three specialist agents run in parallel via shared MCP memory — each reading what the others flagged — then a synthesis agent produces a prioritised action list.
SECURITY AGENT
Hunts injection points, auth flaws, secrets in code, OWASP top 10. Outputs: vulnerability, severity, exact line, fix recommendation.
temp: 0.3 · systematic
PERFORMANCE AGENT
Finds N+1 queries, missing indices, blocking I/O, memory leaks, unoptimised loops. Quantifies: "This loop is O(n²) — at 10K records it will take 45s."
temp: 0.4 · analytical
MAINTAINABILITY AGENT
Flags tech debt — god classes, missing tests, coupling, magic numbers, dead code. Estimates refactor effort in story points.
temp: 0.5 · systematic
ARCHITECT AGENT
Reads what the three specialists flagged and proposes a coherent refactor plan. Prioritises by business risk × implementation cost.
temp: 0.5 · convergent

Research Synthesis — How It Works

Feed Chitti 50 paper abstracts. Agents extract claims, identify conflicts between papers, flag methodology weaknesses, and surface the consensus — and the dissent. The skeptic agent specifically hunts for papers that contradict the mainstream narrative.
EXTRACTOR AGENT
Pulls key claims from each paper, standardises them into structured findings. Identifies effect sizes, sample sizes, and study design.
CONFLICT DETECTOR
Cross-references claims across papers. Surfaces contradictions: "Paper 12 claims X; Paper 31 claims ¬X with 2× sample size."
METHODOLOGY CRITIC
Flags weak methodologies — small n, no control group, industry funding, self-reported data. Weights findings by study quality.
DISSENT HUNTER
Specifically finds the minority view. The papers that didn't get cited. The findings that complicate the consensus. The science that gets ignored.

Crisis Decision Making — How It Works

Under time pressure — server down, PR crisis, financial decision under fire. Four agents run simultaneously. Final output is the fastest safe action, with explicit risk quantification of each option. Designed to run in under 20 seconds on Groq.
OPTIMIST AGENT
Finds the fastest path forward. Assumes best-case. Maps the actions that could resolve this in the shortest time. High temperature — lateral thinking under pressure.
temp: 0.8
PESSIMIST AGENT
For each proposed action, surfaces what could go wrong. Quantifies the downside. Prevents the crisis from becoming a disaster through premature optimism.
temp: 0.4
RISK QUANTIFIER
Assigns probability and impact to each option's failure mode. "Option A: 20% chance of 4h outage vs Option B: 5% chance of data loss." Makes the tradeoff legible.
temp: 0.3
PRAGMATIST AGENT
Reads the risk analysis and picks the highest expected-value action given the constraints. Produces the final recommendation with explicit reasoning.
temp: 0.5
