The 4 architectural innovations — why judges should care

01
🐜
Stigmergic Collaboration
Inspired by ant colonies — agents never talk to each other. They leave ideas in shared MCP memory and read what others left. A creative agent's wild idea modifies the landscape that a systematic agent reads next. This produces emergent intelligence, not orchestrated roleplay.
▸ No existing hackathon project uses stigmergy via MCP
02
🧬
Mid-Run Darwinian Evolution
Every iteration: bottom 30% of agents are pruned based on idea scores. Top 20% become elite and seed the next round. The swarm gets smarter as it runs, not just when you restart it. This is genetic algorithm logic applied to LLM agent selection — in real time.
▸ No other open-source system prunes agents mid-query
03
🧠
MCP as Distributed Working Memory
Model Context Protocol is used not just for tool-calling, but as a shared cognitive whiteboard. Every idea written to it is scored and queryable by all agents. This also means Chitti plugs natively into any MCP-compatible client — Claude Desktop, VS Code — zero integration work.
▸ Novel use of MCP beyond its original design intent

Anticipated judge objections — and why they're wrong

"Multi-agent has already been done — CrewAI, AutoGPT, LangGraph…"
Yes — but those systems have fixed agents. Chitti is the first to apply mid-run natural selection: agents are scored, ranked, and pruned inside a single query. That's not multi-agent. That's an evolving swarm. The difference is what happens between iterations — and no existing system does what Chitti does there.
"The agents still use the same base LLM — how is it really different perspectives?"
Same model, different lenses. Temperature, system prompt, and role create genuinely different reasoning paths — this is validated by cognitive diversity research. A cardiologist and a neurologist using the same brain still disagree on diagnosis. Temperature 0.2 vs 0.9 produces demonstrably different output distributions. The diversity is architectural, not cosmetic.
"What's the real-world application if it takes multiple iterations?"
High-stakes decisions: medical differential diagnosis, legal contract risk, architecture review, research synthesis. These are domains where speed matters less than correctness and where a single LLM's hallucination is dangerous. Chitti's 3–5 iteration run takes under 30 seconds on Groq — faster than calling a second expert.
